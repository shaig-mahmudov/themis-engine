This directory contains the architecture decisions for Themis Engine. Each decision has its own immutable record so it can evolve independently and be linked from code, reviews, and project documentation.

## Conventions

- ADR files use a three-digit sequence and a descriptive kebab-case name.
- Accepted decisions are not edited to hide earlier reasoning. Material changes should be recorded in a new ADR that supersedes the old one.
- Implementation status may be clarified in a record, but the original context, decision, and consequences should remain traceable.
